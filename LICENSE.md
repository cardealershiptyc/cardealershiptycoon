## 🛠️ Troubleshooting Common Issues

**❌ GUI doesn't appear after execution**
The dashboard may fail to render if the game hasn't fully loaded yet.
**Solution:** Wait until you see the dealership map completely load before injecting, then re-execute the **Car Dealership Tycoon Script**.

**❌ Auto Delivery stops mid-session**
The loop can stall if Roblox resets a vehicle's delivery path unexpectedly.
**Solution:** Toggle **Auto Delivery** off and back on using the GUI dashboard — this resets the loop state without needing a full re-inject.

**❌ Unlock All Cars reverts after game events**
Some in-game story triggers reload car lock states.
**Solution:** Enable the **Auto Re-Unlock** option in the GUI (added in v4.6.0) so the **Car Dealership Tycoon Script** automatically re-applies unlocks after cutscenes.

**❌ Speed Changer causes character to clip through floors**
Setting the **Car Dealership Tycoon Script Speed** multiplier above `3.0` causes physics conflicts.
**Solution:** Keep **SpeedMultiplier** at or below `2.5` — the script's built-in limiter will now warn you if you exceed safe values.

**❌ Dupe Car feature not working on Android**
Some Android executors have limited memory access required for duplication.
**Solution:** Ensure your Android executor is updated to the 2026 build — older versions do not support the **Dupe Car** memory bridge used in v4.6.0+.

**❌ Script crashes immediately on injection**
Usually caused by a version mismatch between the executor and the **Car Dealership Tycoon Script**.
**Solution:** Download the latest executor version and re-download the script from the official source above. Clear any cached `.lua` files stored under `C:/Desktop/CDT/cache/`.
